================================================================================
1. QUÉ ES EVSE
================================================================================

App local de edición automática de videos verticales para TikTok / Reels /
YouTube Shorts. Tomas un video (subido o importado de YouTube), eliges un perfil
de marca (logo, imagen "sígueme", video de cierre), y la app aplica un preset
estilo CapCut/Filmora y exporta MP4 9:16 1080×1920 a 60fps con NVENC si tu GPU
lo soporta.

Stack:
- Backend: Flask + threads (sin async). Python 3.10+
- Frontend: HTML + CSS + JS plano, sin framework
- Render: FFmpeg (h264_nvenc o libx264)
- Importación YouTube: yt-dlp con cookies de Chrome/Edge/Firefox/Brave
- Subtítulos: faster-whisper LOCAL (sin OpenAI, sin claves)
- Traducción: deep-translator (Google Translate web, sin clave)

Nombre actual del programa: **Evse** (Ev de Evelin + Se de Sebastian).
  - Versiones descartadas: WorkFast (original), Mailo (muy femenino), Yase.

4. CÓMO ARRANCAR
================================================================================

1. Doble clic en "Abrir WorkFast.bat" (sí, sigue llamándose así por ahora).
2. La primera vez instala Python deps (~250MB): faster-whisper, deep-translator,
   yt-dlp, Flask. Tarda 2-5 minutos según internet.
3. Arranca el server en http://127.0.0.1:5000 y abre el navegador.
4. La PRIMERA vez que generes subtítulos baja el modelo Whisper "small" (480MB).

Variables opcionales en .env:
  WORKFAST_WHISPER_MODEL=small      (tiny / base / small / medium / large-v3)
  WORKFAST_WHISPER_DEVICE=auto      (auto / cuda / cpu)
  WORKFAST_WHISPER_COMPUTE=auto     (auto / int8 / float16)

Smoke test:
  python scripts/smoke_test.py
  → genera un MP4 sintético y lo procesa entero (~10s)

6. BUGS CONOCIDOS / LIMITACIONES
================================================================================

⚠ El sistema Edit/Write de las sesiones de Cowork TRUNCA archivos cuando
  superan ~95-110 KB. Por eso frontend/index.html ha tenido que reconstruirse
  varias veces. Si se va a hacer un cambio grande, usar Python via bash heredoc
  en vez de Edit. El archivo actual es de ~119 KB, peligrosamente cerca del
  límite.

⚠ Si el usuario tiene Chrome cerrado al pulsar "Capturar sesión" de YouTube,
  falla. La app abre Chrome con remote-debugging-port=9222 pero no lo cierra
  después; se queda residente. Idea futura: cerrar el proceso al capturar.

⚠ assets/uploads/ y assets/outputs/ no tienen LRU automático. Crecen sin tope.
  Existe cleanup_outputs() en main.py pero no se llama frecuentemente.

⚠ Si el video original es muy corto (<3s) o sin audio, _strip_silences puede
  comportarse raro. Hay protección contra recortar >70% pero no <3s.

⚠ El botón "Abrir WorkFast.bat" sigue llamándose así. Sería mejor renombrar
  a "Abrir Evse.bat" pero no se ha hecho para no romper accesos directos del
  usuario.

⚠ El Chrome de debug (--remote-debugging-port=9222) tiene puerto fijo y
  sin auth. En localhost no es crítico pero técnicamente cualquier proceso
  local podría leer cookies mientras esa ventana esté abierta.

2. MEJORAR CALIDAD DE VIDEOS (UPSCALING IA)
═══════════════════════════════════════════════════════════════════════════════

OBJETIVO: Toggle "Mejorar calidad" estilo CapCut. Pasa el video por un upscaler
con IA antes del render principal. Output más nítido, colores mejorados, menos
ruido. Visible especialmente en videos viejos o de baja calidad.

OPCIONES EVALUADAS PARA TU PC (GTX 1650 Ti 4GB):

  OPCIÓN A: Real-ESRGAN ncnn-vulkan (RECOMENDADO)
  ----------------------------------------------
  - Binario standalone (~30 MB) que usa Vulkan en GPU
  - NO requiere CUDA Toolkit ni PyTorch (zero deps Python pesadas)
  - Tu GTX 1650 Ti soporta Vulkan perfectamente
  - Modelo `realesr-animevideov3` para video (~12 MB, optimizado para 30/60fps)
  - Modelo `realesrgan-x4plus` para fotorealismo (~64 MB, mejor calidad pero +lento)
  - Velocidad: ~2-4× tiempo real en GTX 1650 Ti con animevideov3
  - URL: https://github.com/xinntao/Real-ESRGAN/releases
  - Plan: descargar zip, extraer en `assets/realesrgan/`, llamar via subprocess
  
  Comando ejemplo:
    realesrgan-ncnn-vulkan.exe -i input.mp4 -o upscaled.mp4 -n realesr-animevideov3
  
  Pero NO procesa video directamente. Hay que extraer frames con ffmpeg,
  procesar cada frame, y reensamblar. Usar `video2x` (Python wrapper) o hacer
  pipeline manual con ffmpeg + Real-ESRGAN.

  OPCIÓN B: Pipeline FFmpeg avanzado (sin IA pero gratis y rápido)
  ----------------------------------------------------------------
  - Filtros: cas (Contrast Adaptive Sharpening) + unsharp + hqdn3d + eq
  - Mejora notable de nitidez y color sin upscaling real
  - 0 deps adicionales, solo cambiar el filter_complex
  - Velocidad: igual que el render normal (sin overhead)
  - Calidad: ~70% del Real-ESRGAN para mejora visible, sin AI real

  Filtros recomendados:
    cas=strength=0.5,
    unsharp=5:5:1.0:5:5:0.0,
    hqdn3d=4:3:6:4,
    eq=contrast=1.05:saturation=1.10:gamma=0.97,
    lutyuv=y=gammaval(0.95)

  OPCIÓN C: Topaz Video AI (PAGO, descartar por ahora)
  ----------------------------------------------------
  - $300 USD, no es opción mientras no haya ingresos

3. INTEGRACIÓN CON FACEBOOK PAGES (PUBLICAR EN FANPAGES)
═══════════════════════════════════════════════════════════════════════════════

OBJETIVO: Vincular cuenta Facebook, listar fanpages del usuario, seleccionar
una para publicar, y programar la subida.

LIBRERÍA: facebook-sdk (oficial Meta) o requests directo a la Graph API.
  pip install facebook-sdk
  
  Mejor: usar requests directo. La SDK oficial está abandonada.

FLUJO TÉCNICO:
1. Usuario entra a "Conectar Facebook" en la app
2. Se abre OAuth de Facebook en navegador (con scopes: pages_show_list,
   pages_manage_posts, pages_read_engagement)
3. Callback → guardamos access_token de USUARIO en assets/library/facebook.json
4. Endpoint /me/accounts devuelve las pages con sus PAGE_ACCESS_TOKEN
5. Usuario selecciona una page del dropdown
6. Para publicar:
   POST https://graph.facebook.com/v21.0/{page_id}/videos
   body: file=video.mp4, description=texto, scheduled_publish_time=timestamp,
         published=false (para programar)

REQUISITOS:
- Crear app en developers.facebook.com (gratis)
- App ID + App Secret van a .env (NO commiteados)
- Verificar la app para producción O usar modo "desarrollo" con max 25 testers
- Para video, el usuario debe tener role "Editor" o superior en la page

PROGRAMACIÓN:
- Endpoint /scheduled_posts en la API
- scheduled_publish_time = timestamp UNIX (mínimo +10 minutos en el futuro)
- Máximo 6 meses adelante
- Se programa de forma "non-published" hasta que llegue la hora

ARCHIVOS A CREAR:
- backend/social/facebook.py: cliente, OAuth, publish
- frontend: nueva sección "Programar publicación" con dropdown de pages,
  date+time picker, preview del post

ALTERNATIVA SIN OAUTH (más simple, menos pulida):
- El usuario pega su PAGE_ACCESS_TOKEN long-lived directamente
- Le explicamos cómo generarlo en developers.facebook.com → Tools → Graph API Explorer
- Más fácil de implementar (zero OAuth flow), peor UX

4. DESCRIPCIONES + HASHTAGS GENERADOS POR IA (OLLAMA LOCAL)
═══════════════════════════════════════════════════════════════════════════════

OBJETIVO: Botón "Generar descripción y hashtags" que toma el título del video
(o transcripción si está disponible), llama a Ollama local, y devuelve:
  - Descripción de 2-3 líneas atractiva para TikTok/Reels
  - 5-8 hashtags virales del nicho
  - Hook de primera línea para engagement

OLLAMA: corre LLMs localmente, sin API keys, sin internet después de descargar.
  - Instalar: https://ollama.com/download (instalador para Windows)
  - Una vez instalado, hace `ollama pull <modelo>` y queda en background
  - Endpoint local: http://localhost:11434/api/generate

MODELOS RECOMENDADOS PARA TU PC (GTX 1650 Ti 4GB + 15GB RAM):

  RECOMENDADO: Llama 3.2 3B
  --------------------------
  - Tamaño: ~2 GB descargado, ~2.5 GB VRAM en uso
  - Calidad: muy buena para textos cortos (descripciones, hashtags, hooks)
  - Velocidad: 30-50 tokens/seg en tu GPU (rapidísimo)
  - Comando: ollama pull llama3.2:3b
  - Ideal para: descripciones, hashtags, traducciones cortas

  ALTERNATIVA: Phi-3 Mini (3.8B)
  ------------------------------
  - Microsoft, optimizado para razonamiento
  - Tamaño: ~2.3 GB
  - Calidad: excelente para tareas estructuradas (JSON, listas)
  - Velocidad: ~25-40 tok/s
  - Comando: ollama pull phi3:mini

  ALTERNATIVA: Qwen 2.5 3B
  ------------------------
  - Alibaba, multilingüe (excelente español)
  - Tamaño: ~2 GB
  - Comando: ollama pull qwen2.5:3b

  EVITAR (no caben bien en 4GB VRAM):
  - Llama 3.1 8B (necesita 6+ GB)
  - Mistral 7B (necesita 5+ GB)
  - Llama 3.3 70B (imposible)

PROMPT EJEMPLO:

     'description': '<2-3 líneas en español, tono casual>',
     'hashtags': ['<8 hashtags relevantes y virales del nicho>']
   }"
